Service-by-Service
Monthly Itemisation — 2 Cameras
ServiceMonthly CostNotes
S3 PUTs — HLS segments $1.80 hls_time=10s → 6 segments/min × 2 cams × 43,200 min/month = ~518,400 PUTs. $0.0035/1,000 PUTs = $1.81. Dominant cost driver.
S3 Storage — clips (7-day rolling) $0.52 ~2 cameras × 720Kbps avg × 300s/clip × ~2,016 clips in 7 days ≈ ~38GB at $0.023/GB. S3 lifecycle deletes after 7 days.
S3 Storage — live HLS (1-day rolling) $0.06 ~5 × 10s segments × 2 cams × ~2.5MB each = <1GB/day rolling window. Negligible.
CloudFront data transfer out $0.20 Estimated ~2GB/month dashboard usage (HLS playback + clip browsing). First 1TB free tier offsets most.
Lambda — 3 functions $0.00 Webhook: ~500 events/month. Clip/Events API: ~200 calls/month. <100MB-seconds total. Free tier: 400,000 GB-seconds/month.
API Gateway — HTTP API $0.00 ~700 requests/month (dashboard API + webhooks). Free tier: 1M requests/month.
DynamoDB — cgm-events table $0.00 ~500 events/month × ~1KB = <1MB storage. Reads/writes negligible. Free tier covers entirely.
AWS IoT Core — credential provider $0.00 Pi refreshes credentials hourly. ~720 requests/month. IoT credential provider: $0.02/million requests.
SNS — SMS alerts ~$0.00 Transactional SMS: ~$0.00645/message to Canada. 0–5 alerts/month = <$0.04. Negligible.
SSM Parameter Store $0.00 4 standard parameters. Standard tier is free. Reads: ~500/month (webhook Lambda). Free.
Cognito User Pool $0.00 1 MAU (invite-only). Free tier: 50,000 MAUs/month permanently.
Total — 2 cameras ~$2.58 On-demand pricing · 24/7 recording · 7-day clip retention
S3 PUTs are the only meaningful cost. Each HLS segment write = 1 PUT request. With hls_time=10 already set in stream.sh, this is already optimised. Increasing hls_time to 15 would reduce PUTs by 33% (~$0.60 saving) at the cost of slightly higher live stream latency.

Optimisation
Reducing the Bill
LeverSavingTrade-off
Increase hls_time from 10s → 15s −$0.60/mo Live stream latency increases from ~30s to ~45s. Still acceptable for home monitoring.
Reduce clip resolution (1080p → 720p) −$0.15/mo Lower storage. Reduce ffmpeg bitrate in stream.sh (-b:v 800k). TAPO C100 streams 1080p natively.
Enable S3 Intelligent-Tiering on clips/ ~$0.00 Objects accessed <30 days — Intelligent-Tiering is not beneficial at 7-day TTL. No saving.
Use S3 Express One Zone for live/ prefix +cost Express One Zone charges per-request; at HLS segment volume this would cost more. Not recommended.
Schedule Pi to only stream during home-away hours up to −50% If streaming ~12hrs/day instead of 24hrs, halves all S3 costs. Modify stream.sh with time check or use systemd timer.
